Introduction

For this assignment I decided to create the basics of a new startup called chattr. Chattr allows users to create a video chat room and then have conversations with their friends.

Highlights

  • Working webcam input, including the ability to add CSS filters to the webcam input: grayscale, inverted and sepia.
  • The basics of WebRTC communication has been setup including importing the Socket.IO library. To complete this there needs to be some communication with the server before the peer-2-peer communication can occur.
  • JQuery email form validation. I found a Regex email checking string on www.stackoverflow.com but wrote all the other JQuery code in index.js myself, using the documentation and Google searches to find method names.
  • CSS animation effects using the Animate.css and Wow.js frameworks. Animate.css contains the CSS3 animations and the Wow.js library is a javascript library that calls these animations when the elements are scrolled into the viewport.
  • Fallback pages for browser's that don't have WebRTC available. The server also serves a 404 page when an unavailable resource is requested.
  • Procedurally generated white noise while waiting for video input to be allowed

HTML, CSS and Javascript

I've used HTML tags in an appropriate way, utilising new tags such as <section> and <nav> rather than simply having a <div>. All HTML, CSS and Javascript are separate from each other except an Adobe TypeKit call. Adobe TypeKit is a CDN for fonts and the line of Javascript authenticates itself, before adding the relevant @font-face CSS styles.

There is one commented section of Javascript in conversation.js that generates a series of 20 random white noise images. These are then used as loading image whilst webcam input is being approved. This can be enabled by uncommenting lines 16, 42 and 96. The reason it's disabled is because it was causing a render blocking issue that I was unable to fix in time, but an image of the white noise can be seen here.

I used the Gridism.css framework to help structure the layout of the HTML. This is a minamalistic CSS framework that comes with grid size definitions that also gracefully scale using @media-queries

Images and graphics

  • Adobe Illustrator was used to create the scroll down image and the log out image. I chose to use Adobe Illustrator as I've had experience with it in the past, and so felt I'd be able to achieve more in the assignment if I didn't have to learn how to use another piece of software.
  • Adobe Photoshop was used to carry out some minor adjustments to images. For example, the Redis logo text needed to be in white rather than gray to match the background. I therefore created a selection box over the text and added a colour overlay. This also applied to the Node.js logo. Photoshop was also used to prototype with colour schemes. After I'd drawn up mockups of the website design, I imported them into photoshop and roughly coloured in the different elements to see how they looked. I decided it was best to try and do most of the design work using CSS3 rather than images to decrease the number of resources that need to be loaded. This is why the contact images where rounded and had borders added using CSS.

Deployment

The site is running on a Node.js server using Heroku, and this is why there is a Procfile package.json file in the directory. The site uses the Express.js framework to and the server can be found in server.js. This serves the static files, redirects to a 404 page if a requested resource can't be found and makes sure the Content-Type is application/xhtml+xml

Standards and Testing

My website serves XHTML 5 and is fully compliant to the standards. It's also fully compliant to the CSS3 standards, apart from cases where I've used browser prefixes such as -webkit, -moz and -o. In all cases I've also included the standardised version of the property. I've tested the website on the following browsers:

  • Google Chrome Version 33.0.1750.152
  • Safari Version 7.0.2 (9537.74.9)
  • Firefox 27.0.1
